## 2. 网站是怎么跑起来的（简单原理）

```
访问者输入 hustsv.org
      │
      ▼
GoDaddy 的 DNS 记录 → 告诉浏览器去 GitHub 的服务器
      │
      ▼
GitHub Pages → 把网页文件发回去，浏览器显示
```

- 你把网页文件放进 GitHub 仓库，GitHub 自动把它变成网站。
- 域名和网站是**两套系统**：域名在 GoDaddy，网站在 GitHub，中间靠 DNS 记录连接。
- HTTPS 证书（那个小锁）由 GitHub 自动签发和续期，不用管。

## 3. 怎么改网站内容（宣传部门最常用）

有两种方式，**不会命令行也能改**：

### 方式 A：在 GitHub 网页上直接改（最简单，推荐日常用）
1. 打开对应仓库（如 `github.com/hust-svca/hustsv`）。
2. 点进要改的文件（见下面"改什么改哪里"）。
3. 点右上角**铅笔图标 ✏️** 编辑。
4. 改完，页面底部点 **Commit changes**（填一句说明，比如"更新理事会名单"）。
5. 等约 1 分钟，网站自动更新。刷新页面即可看到（看不到就 Cmd+Shift+R 强制刷新）。

### 方式 B：本地改 + 推送（懂技术的人用）
```bash
cd hustsv-site        # 或 hustsvfund-site
# 用编辑器改文件
git add -A && git commit -m "说明改了什么" && git push
```

### 改什么，改哪里

**校友会官网 `hustsv.org`（仓库 `hust-svca/hustsv`）**
| 想改的东西 | 改这个文件 |

### 加图片
网页里的图片放在各仓库的 `assets/` 文件夹里。加图步骤：把图片文件传进 `assets/`，然后在文章数据里写路径 `../assets/文件名.jpg`。图片建议先压到宽度 900 像素以内（省流量、加载快）。

## 4. 改完怎么确认上线了

- 每次 push（或网页上 Commit）后，GitHub 自动部署，**约 1 分钟**。
- 直接刷新网站看效果（强制刷新 **Cmd+Shift+R** 清缓存）。
- 想看部署状态：仓库页面 → **Actions** 标签，绿勾=成功，红叉=失败。
- **偶尔 GitHub 部署会失败**（显示 "try again later"）——这不是你的错。解决办法：在 Actions 里点 **Re-run jobs**，或做一次空改动重新触发即可。

## 6. DNS 记录说明（在 GoDaddy 里，别乱删）

在 GoDaddy → 域名 → DNS 里能看到一堆记录，分三类：

- **网站相关（连接到 GitHub）**：4 条 `A @ → 185.199.108–111.153` + `CNAME www → hust-svca.github.io`。**这些是网站的命根子。**
- **邮箱相关（微软 365，仅 hustsvfund.org）**：`MX`、两条 `TXT`（含 SPF）、`autodiscover`、`msoid`、`sip`、`lyncdiscover`、两条 `SRV`。**删了域名邮箱就收不到信了，千万别动。**
- **GoDaddy 基础设施**：`NS`、`SOA`、`_domainconnect`、`ftp` 等，系统用的，别动。

> 一句话：**网站走 A/CNAME，邮箱走 MX/TXT/SRV**，互不影响。非必要不碰 DNS。

## 7. 技术细节附录（给懂技术的志愿者）

**本地开发**：两个站分别在 `hustsv-site/`、`hustsvfund-site/` 目录，都是纯静态站。改完 `git push` 即自动部署。

**校友会站的文章是怎么来的**：`data/articles.js` 里的 56 篇活动报道，是用源文件夹里的两个脚本从微信公众号导出文件生成的：
- `extract-images.js`：从 `.mhtml` 里提取文章图片、压缩、存到 `assets/`。
- `build-articles.js`：解析 `.md` 文章正文、套用本地图片路径、生成 `data/articles.js`。
- 加新活动文章时：把公众号文章的 `.md` + `.mhtml` 放进源文件夹，重跑这两个脚本，再 push。
> 注意：微信图床有防盗链，图片必须自托管（脚本已处理），不能直接用微信图片链接。

**基金会站**：`data/news.js` 是**手工维护**的（不是脚本生成），直接编辑即可。文章图片来自 Wuhan United 的 Google 站，已下载自托管在 `assets/`。

**设计系统**：两个站共用同一套视觉风格（深蓝 + 铜色 + 等宽字体标签），CSS 在各自的 `style.css`。

**换新仓库/迁移**：GitHub Pages 免费版要求仓库 **Public**；自定义域名靠仓库根目录的 `CNAME` 文件 + GoDaddy 的 DNS 指向。

## 8. 出问题怎么办

| 现象 | 排查 |
|---|---|
| 网站打不开（404） | 大概率 GitHub Pages 没开或部署失败：仓库 Settings → Pages 确认已启用；Actions 里看部署是否红叉，重试。 |
| 改了没生效 | 浏览器缓存，强制刷新 Cmd+Shift+R；或部署还没跑完，等 1 分钟。 |
| 整个域名打不开 | 查 GoDaddy 域名**是否过期**；`dig hustsv.org +short` 应返回 4 个 185.199.x.x。 |
| 邮箱收不到信 | 检查 GoDaddy DNS 里的 MX/TXT 记录是否被误删。 |
